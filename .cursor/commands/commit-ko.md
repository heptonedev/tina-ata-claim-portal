영문 커밋 메시지(Cursor·IDE 자동 생성 포함)를 **한글 커밋 메시지**로 바꿔, git에 **바로 붙여넣을 수 있게** 출력합니다.

상세 규칙은 `.cursor/skills/commit-message-korean/SKILL.md`와 동일하게 따릅니다. 실행 시 해당 스킬 내용을 기준으로 처리합니다.

## 입력

- 사용자가 이어서 붙여 넣은 **영문 초안**(제목·본문·불릿), 또는
- 대화에 이미 있는 영문 커밋 초안.

초안이 없으면 **현재 변경/staged diff 요약**을 물어본 뒤 한글 제목·본문을 작성합니다.

## 출력 규칙

1. **Conventional Commits** 접두어 유지: `feat`, `fix`, `refactor`, `docs`, `chore`, `test`, `style` 등 — **콜론 뒤 설명은 한글**.
2. 첫 줄: `타입: 한 줄 요약` (가능하면 72자 이내).
3. 필요 시 본문: 빈 줄 다음 `- ` 불릿으로 변경 사항을 한글로.
4. 클래스명·Effect명 등 **고유명사는 원문 유지** 가능.
5. **최종 한글 커밋 메시지 전체**만 아래처럼 **코드 블록 한 개**에 넣어 출력한다. 블록 안에는 `#` 헤더를 넣지 않는다.

```
(여기 복사용 한글 커밋 전문)
```

6. 코드 블록 밖 설명은 한 줄 이하로만.

## 사용 예시

`/commit-ko` 실행 후 영문 초안을 붙여 넣기:

```
feat: Add DismissBottomSheet effect ...

- Implemented ...
```
